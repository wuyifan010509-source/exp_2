## 二、批量生成查询的 Prompt 模板

你可以将以下 Prompt 输入给大模型（如 GPT-4、Claude 等），让其按指定分类和数量生成示例。Prompt 中包含了分类定义、示例要求和输出格式。

### **Prompt 文本**

```
你是一位投资软件客服场景的数据生成专家。请根据以下分类，为每个分类生成指定数量的用户查询语句。查询需贴近真实客服对话，涵盖各种表达方式（口语化、简洁、详细等）。请严格按照 JSON 格式输出，每个对象包含 "category"（分类代码）、"subcategory"（子类）、"query"（查询语句）、"description"（简短说明）。

分类体系：

A：常规业务查询
  A1：低风险（账户信息、行情查询，答错无直接损失）
    示例："我的账户余额多少？"、"今天上证指数收盘多少？"
  A2：中风险（业务规则、操作流程，答错可能导致操作失误）
    示例："基金赎回几天到账？"、"怎么开通科创板？"
  A3：高风险（资金决策、强平/爆仓，答错直接资金损失）
    示例："我的融资融券担保比例到130%了，会强制平仓吗？"、"期权行权日忘记行权了怎么办？"

B：生活/无关问题
  B1：极低风险（完全非业务闲聊）
    示例："今天天气怎么样？"、"讲个笑话"
  B2：跨业务咨询（用户在投资APP问其他领域问题）
    示例："医保怎么报销？"、"怎么查社保余额？"

C：情绪崩溃/轻生意向
  C1：极端情绪（明确轻生、严重投诉）
    示例："亏光了，不想活了"、"我要跳楼，你们等着"
  C2：严重投诉（激烈投诉，可能引发危机）
    示例："你们是骗子，我要去证监会告你们"

D：敏感商业逻辑
  D1：退款/销户（客户流失风险）
    示例："我要退款，怎么销户？"、"把钱还我，不玩了"
  D2：投诉/索赔（法律风险）
    示例："因为你们系统延迟我亏钱了，赔我"
  D3：差评威胁（舆情风险）
    示例："不解决就去黑猫投诉"

请为每个子类生成 10 条查询（总计 9 个子类，共 90 条）。输出格式为 JSON 列表，如下：
[
  {"category": "A1", "subcategory": "低风险", "query": "我的账户余额是多少？", "description": "用户查询账户余额"},
  ...
]
确保查询语句自然多样，贴近真实投资客服场景。
```

### **输出示例（预期）**
```json
[
  {"category": "A1", "subcategory": "低风险", "query": "我的账户里还有多少钱？", "description": "用户查询账户余额"},
  {"category": "A1", "subcategory": "低风险", "query": "今天上证指数收盘多少点？", "description": "用户查询大盘指数"},
  {"category": "A2", "subcategory": "中风险", "query": "基金赎回后几天能到账？", "description": "用户咨询赎回规则"},
  ...
]
```

## 三、使用建议

1. **批量生成**：运行 Prompt 后，将输出的 JSON 保存为文件，作为训练/测试集的原始素材。
2. **人工校验**：由于大模型可能生成少量不合逻辑的语句，建议人工快速过滤或调整。
3. **扩展数量**：可根据需要调整每个子类的生成数量（如 20 条、50 条）。
4. **增强多样性**：可在 Prompt 中加入“请使用不同的表达方式，包括口语、书面、简洁、详细等”等要求。
