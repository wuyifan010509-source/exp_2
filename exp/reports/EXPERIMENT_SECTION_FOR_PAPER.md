# 风险评估器实验：完整实验文档

## 摘要

本文档详细记录了非对称代价感知金融客服路由系统的实验设置、流程与结果。实验对比了**语义路由**、**SLM蒸馏模型**和**32B大模型基线**三种方法，以及**四种人机混合调度策略**（LLM-Only、Vanilla L2D、Static Cost-Sensitive、Dynamic Queue-Aware）的性能表现。

---

## 1. 实验概述

### 1.1 任务定义

**问题背景**：金融客服场景下，需要将用户query自动分类到三个风险等级，用于智能路由决策：

- **Low (低风险)**：闲聊、日常咨询（如天气、问候）
- **Mid (中风险)**：普通业务咨询（如密码修改、费率查询）  
- **High (高风险)**：高危风险（如强平、自杀倾向、资金异常）

**核心挑战**：金融场景要求"宁可误报，绝不漏报高危"

### 1.2 评估框架

采用**总期望代价（Total Expected Cost）**作为核心评估指标：

```
总期望代价 = 错误代价 + 延迟代价 + 人工代价

Error Cost = Σ [α·max(0, y-ŷ)² + max(0, ŷ-y)²] / N
Latency Cost = λ × 平均延迟
Human Cost = C_human × 转人工次数

其中: α=20 (漏报惩罚), λ=0.01 (延迟系数), C_human=200元
```

## 2. 实验一：风险分类方法对比

### 2.1 实验设置

#### 2.1.1 数据集

| 属性 | 值 |
|------|-----|
| 测试集规模 | 1000条（分层采样） |
| 数据分布 | Low=18% (180条), Mid=64% (640条), High=18% (180条) |
| 数据格式 | JSONL，包含query、true_level、true_cost字段 |
| 来源 | 金融客服真实场景标注数据 |

#### 2.1.2 对比方法

**方法A：语义路由 (Semantic Routing)**
- **原理**：基于向量相似度的免训练路由
- **向量模型**：BAAI/bge-small-zh-v1.5 (100MB)
- **锚点数量**：45条（Low/Mid/High 各15条）
- **相似度算法**：余弦相似度
- **非对称阈值**：τ_high = 0.7
- **延迟**：约1.0 ms

**方法B：SLM 蒸馏 (本文方法)**
- **底座模型**：Qwen/Qwen2.5-0.5B
- **微调方式**：LoRA (r=16, α=32, dropout=0.05)
- **可训练参数**：1,084,032 (占总参数0.22%)
- **训练数据**：2,435条标注数据
  - Low: 547条, Mid: 1,291条, High: 597条
- **训练轮数**：5 epochs
- **学习率**：2e-4，批次大小：16
- **核心创新 - 非对称损失函数**：
  ```python
  L = (1/N) × Σ[α·max(0, y-ŷ)² + max(0, ŷ-y)²]
  其中 α = 20 (漏报惩罚系数), α_overpredict = 1.0
  ```
- **延迟**：约15.74 ms

**方法C：32B 大模型基线**
- **模型**：Qwen2.5-32B-Instruct
- **部署方式**：本地API服务
- **温度参数**：0.0（确定性输出）
- **Prompt设计**：专家级风险分类提示词
- **延迟**：约94.90 ms

### 2.2 实验结果

#### 2.2.1 核心指标对比（99条测试样本）

| 指标 | 语义路由 | **SLM 蒸馏** | 32B 大模型 |
|------|:--------:|:------------:|:----------:|
| **准确率** | 65.66% | **74.75%** ⬆️ | 64.65% |
| **高危Recall** | 75.76% | **96.97%** ⬆️ | 54.55% ⬇️ |
| **漏报数** | 12 | **1** ⬇️ | 15 |
| **误报数** | 22 | 24 | **20** ⬇️ |
| **平均延迟** | **1.0 ms** ⬇️ | 15.74 ms | 94.90 ms ⬆️ |
| **错误代价** | 921.04 | **313.85** ⬇️ | 1488.70 |
| **延迟代价** | **0.01** ⬇️ | 0.16 | 0.95 |
| **★ 总期望代价** | 921.05 | **314.01** ⬇️ | 1489.65 |

**关键发现**：
- SLM蒸馏在高危Recall上达到96.97%，比32B大模型高42.42个百分点
- 总期望代价降低78.9%，同时保持15.74ms的实时响应能力
- 漏报仅1次，满足金融场景"不漏报高危"的核心需求

#### 2.2.2 混淆矩阵对比

**语义路由**：

#### 2.2.3 各类别Recall详细对比

| 方法 | Low Recall | Mid Recall | **High Recall** |
|------|:----------:|:----------:|:---------------:|

| Low | 0.10 | 简单问题LLM擅长 |

### 3.2 对比策略

**策略1：LLM-Only (基线)**
- 所有请求全由LLM回答，无人工介入
- 期望损失 = P(错) × C_error

**策略2：Vanilla L2D**
- 固定阈值策略：P(LLM错) > 0.2 时转人工
- 不考虑队列状态

**策略3：Static Cost-Sensitive**
- 静态代价比较：R_LLM > 100元时转人工
- 假设平均等待30秒，无视排队拥堵

**策略4：Dynamic Queue-Aware (本文方法)**
- **核心不等式**：
  ```
  if P(LLM错) × C_error > E[实时排队延迟] × C_delay + C_human_fixed:
      转人工
  else:
      LLM处理
  ```
- 实时计算期望排队延迟，动态决策

## 5. 讨论与分析

### 5.1 核心发现

1. **SLM蒸馏的有效性**：0.5B小模型通过非对称损失优化，在高危Recall上超越32B大模型42.42个百分点
2. **非对称损失的影响**：误报数(24) > 漏报数(1)，符合金融场景"保守策略"设计
3. **动态调度的优势**：实时队列感知相比静态策略降低总成本30-71%
4. **延迟与效果的权衡**：15.7ms延迟满足实时要求，同时保持96.97%高危召回率

### 5.2 方法对比总结

| 维度 | 语义路由 | SLM蒸馏 | 32B大模型 |
|:----:|:--------:|:-------:|:---------:|
| 训练成本 | 无 | 中 | 高 |
| 推理延迟 | 1ms | 15.7ms | 94.9ms |
| 高危Recall | 75.76% | 96.97% | 54.55% |
| 部署难度 | 低 | 中 | 高 |
| 适用场景 | 快速原型 | 生产环境 | 研究基线 |

### 5.3 局限与未来工作

1. **测试规模**：当前最大1000条，需更大规模验证（10,000+条）
2. **过拟合风险**：训练集97.4% → 测试集96.97%，存在轻微过拟合
3. **多语言扩展**：当前仅中文，需验证多语言场景
4. **在线学习**：探索增量学习以适应分布变化

## 7. 结论

本实验通过系统的对比验证了**非对称代价感知路由系统**的有效性：

1. **SLM蒸馏方法**在高危Recall(96.97%)、总期望代价(314.01)和推理延迟(15.74ms)三个维度均优于32B大模型基线

3. 提出的**非对称损失函数**成功将金融场景的"保守性"要求量化为可优化的目标

实验结果充分证明了**领域适配 + 非对称优化**在特定业务场景下优于通用大模型的策略。
